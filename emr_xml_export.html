核心政策變更

衛福部公告之「救急救難一站通」計畫，針對四大重症之交換機制已由 EEC 線上交換 轉為 XML 實體檔案匯出。

開發因應目標

  • 自動化排程執行匯出任務
  • 產出符合 EEC 規格之 XML
  • 提供 EMR 1.0 / 2.0 雙軌解決方案

業務端對應指南

Business

如何回覆醫院客戶？

Q: 什麼時候能配合一站通？
模組預計 3 月中開發完成，配合訂單流程建議報予客戶 2 個月內上線。
Q: 我們是 1.0 版可以裝嗎？
可以，不論是 EMR 1.0 或 2.0，我們均有提供專屬模組來支援 XML 匯出。

業務端行動清單

  • 系統版本確認： 客戶 EMR 為 1.0 或 2.0。
  • 路徑規劃： 確認一站通軟體抓取 XML 的指定目錄。
  • 範圍定義： (多院區客戶) 確認當次需求院區範圍。
